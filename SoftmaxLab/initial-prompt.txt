1. Цель и контекст
Дашборд для визуализации работы функции softmax в языковых моделях. Я хочу оценить, насколько красивый UI можешь сделать. Поэтому уделяй особое внимание визуальной полировке: типографике, отступам, тонким теням, скруглениям, плавным анимациям при перерисовке.
Это интерактивный инструмент: пользователь меняет логиты и параметры (температура, top-k, top-p), а график вероятностей пересчитывается в реальном времени.
2. Стек

Чистый HTML + CSS + JavaScript, графики на Chart.js 4
Chart.js подключи через CDN: https://cdnjs.cloudflare.com/ajax/libs/Chart.js/4.4.0/chart.umd.min.js
Один файл index.html, открывается двойным кликом в браузере
Никакого React, Vue, сборщиков, npm

3. Стиль
Светлый, лёгкий, футуристичный, динамичный. Конкретно:

Фон страницы: #f9fafb или близкий
Карточки/панели: белые, скругление 12–16px, тень 0 1px 2px rgba(0,0,0,0.04)
Шрифт: системный sans-serif (-apple-system, BlinkMacSystemFont, "Inter", sans-serif)
Акцентный цвет: что-то холодное и техническое (например, фиолетовый #8b5cf6 или бирюзовый #06b6d4)
Любая перерисовка — с плавной анимацией (200–400 мс)
Никаких эмодзи в качестве иконок, только SVG
Без градиентов, без drop-shadow, без неона

4. Структура страницы (сверху вниз)
Блок 1 — управление количеством логитов

Горизонтальный ползунок от 2 до 100
Слева подпись «Количество логитов», справа — текущее значение
При изменении количества логитов:

Старые значения сохраняются
Новые появляются справа со значением = половина текущего вертикального масштаба графика логитов (то есть, если максимум среди существующих логитов 100, новые получают значение 50; если максимум 800 — то 400)
При уменьшении количества — логиты справа просто исчезают, их значения нигде не сохраняются


Рядом с этим блоком — три кнопки:

«Случайные» — заполнить логиты случайными значениями от 0 до 1000
«Все 0» — обнулить все логиты
«Все 1» — установить все логиты в 1



Блок 2 — график логитов

Столбчатая диаграмма (Chart.js, type: bar)
Логиты — числа от 0 до 1000, с точностью до сотых (две цифры после запятой)
Вертикальный масштаб автоматический: самый большой логит занимает всю высоту графика. Если все равны 0 — пусть «1» занимает всю высоту (минимальная единица масштаба).
Клик по столбцу открывает поле ввода (можно через prompt() или нативный <input>-оверлей — выбери что красивее), куда можно ввести новое значение этого логита (0–1000, до сотых). После ввода график перерисовывается, как и график вероятностей под ним.
Тултипы при наведении показывают точное значение логита

Блок 3 — параметры softmax
Расположение: горизонтальная панель.
Слева — температура:

Один горизонтальный ползунок, диапазон 0.01 — 5.0, шаг 0.01
Рядом — поле ввода числа (синхронизировано с ползунком: меняешь одно — меняется другое)
Подпись «Температура»

Справа — top-k и top-p, два поля одно над другим:

Top-k: чекбокс слева + поле ввода целого числа от 1 до N (где N — текущее количество логитов). Когда чекбокс снят — top-k не применяется, поле визуально приглушено.
Top-p: чекбокс слева + поле ввода числа от 0.01 до 1.0 с шагом 0.01. Когда чекбокс снят — top-p не применяется.

Блок 4 — график вероятностей

Столбчатая диаграмма точно под графиком логитов, с тем же горизонтальным масштабом — каждая вероятность находится строго под своим логитом
Вертикальный масштаб автоматический: самая большая вероятность занимает всю высоту
Если включены top-k и/или top-p: обнулённые токены отрисовываются серым полупрозрачным столбцом, а проходящие через фильтр — основным акцентным цветом
Тултипы при наведении показывают точное значение вероятности (до 4 знаков после запятой)

5. Логика расчётов
Формула в каждой перерисовке:

Берутся текущие логиты L = [l₁, l₂, ..., lₙ]
Применяется температура: L' = L / T
Вычисляется softmax: pᵢ = exp(l'ᵢ) / Σexp(l'ⱼ)
Если включён top-k: оставляем k самых вероятных, остальным p = 0
Если включён top-p: сортируем по убыванию, берём минимум, чья кумулятивная сумма ≥ p, остальным p = 0
После top-k/top-p пере-нормализуем (среди оставшихся) так, чтобы сумма = 1
Рисуем все вероятности (включая нули — серым)

Численная стабильность: при экспоненциировании используй стандартный трюк — вычитай максимум из L' перед exp, чтобы избежать переполнения при больших значениях.
6. Стартовое состояние
При первой загрузке страницы:

50 логитов, значения случайные от 0 до 1000
Температура: 1.0
Top-k: отключён
Top-p: отключён

7. Поведение

Любое изменение любого контрола (ползунок, чекбокс, поле ввода, клик по столбцу, кнопка) → немедленная перерисовка обоих графиков с плавной анимацией
Клик по столбцу графика вероятностей ничего не делает (он только для отображения, не редактируется)
Поля ввода top-k и top-p должны валидировать значения (не уходить за границы)
Ползунок температуры и поле ввода температуры должны быть синхронизированы
